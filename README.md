# 🏢 Proyecto de Gestión de Empleados ATTUNE

## 🎯 **Propósito del Proyecto**

Este proyecto busca **desarrollar una aplicación multiplataforma para la gestión de empleados**. El objetivo general es **optimizar la gestión de personal y procesos** en las PYMES, utilizando tecnologías que permitan la centralización, la automatización y la facilidad de acceso a la información. Esto tiene el fin principal de **reducir el tiempo dedicado a tareas administrativas** en los primeros seis meses de su implementación.

## ✨ **Objetivos Clave**

El desarrollo se enfoca en cumplir los siguientes objetivos específicos:

*   **Usabilidad y Satisfacción:** Desarrollar una interfaz que cumpla con el 100% de los requisitos de usabilidad y alcance un índice de satisfacción del usuario del **85%** en pruebas de usabilidad.
*   **Centralización de la Información:** Mejorar la eficiencia mediante la integración de todos los datos del personal en una única base de datos, lo que **reducirá errores de duplicidad** y el tiempo de búsqueda.
*   **Seguridad:** Implementar **medidas de seguridad robustas** para proteger datos sensibles y **superar las pruebas de penetración con 0% de vulnerabilidades críticas**.

## 📄 **Documentación del Proyecto**

La documentación completa del proyecto está organizada por fases y es esencial para entender el alcance, diseño y mantenimiento del sistema.

### Fase de Planificación (I)
*   **Acta de Constitución del Proyecto (Project Charter):** Define alcance, stakeholders clave, roles, responsabilidades, hitos y criterios de éxito.
*   **Estudio de Viabilidad:** Incluye análisis técnico, operativo, económico, legal y regulatorio, además de riesgos y mitigaciones.
*   **Plan de Gestión del Proyecto:** Detalla la gestión de alcance, tiempo (cronograma), costos, calidad, recursos humanos, comunicaciones, riesgos, adquisiciones y stakeholders.

### Fase de Requisitos (II)
*   **Especificación de Requisitos de Software (ERS / SRS):** Contiene requisitos funcionales y no funcionales (seguridad, rendimiento, usabilidad), casos de uso y modelos de datos.
*   **Prototipos y Maquetas:** Incluye diseños de interfaz de usuario (UI/UX) y flujos de usuario.

### Fase de Diseño (III)
*   **Documento de Diseño de Software (DDS / SDD):** Describe la arquitectura del sistema, el diseño de componentes (módulos, clases), el diseño de la base de datos (esquema, relaciones), APIs, estrategias de seguridad y despliegue.
*   **Diagramas de Diseño:** Incluye diagramas UML (clases, secuencia, componentes, despliegue) y diagramas de flujo de datos.

### Fase de Pruebas (V)
*   **Plan de Pruebas:** Estrategia, tipos de pruebas (unitarias, integración, sistema, aceptación) y entorno.
*   **Casos de Prueba:** Identificador, descripción, precondiciones, pasos de ejecución y resultados esperados.
*   **Informes de Pruebas:** Resultados, defectos encontrados, cobertura de pruebas y conclusiones.

### 📘 **Uso y Operación del Sistema (Fase de Despliegue y Mantenimiento VI)**

*   **Manual de Usuario:** Guía de instalación, **instrucciones de uso del sistema**, FAQ y solución de problemas comunes.
*   **Manual de Operaciones/Administración:** Procedimientos para la instalación en producción, backup/recuperación, monitoreo, gestión de errores, logs y procedimientos de actualización.
*   **Notas de la Versión (Release Notes):** Documenta nuevas características, mejoras, correcciones de errores y problemas conocidos.
*   **Plan de Mantenimiento:** Define tipos de mantenimiento (correctivo, adaptativo, perfectivo), procedimientos de soporte y Acuerdos de Nivel de Servicio (SLA).
